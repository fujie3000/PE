## 链式思维提示

<Screenshot src={COT} alt="COT" />

图像来源：[Wei et al. (2022)](https://arxiv.org/abs/2201.11903)

链式思维提示是在中间推理步骤中实现复杂推理能力的一种方法，由[Wei et al. (2022)](https://arxiv.org/abs/2201.11903)引入。您可以将其与少样本提示结合使用，以在需要先进行推理然后回答的更复杂任务上获得更好的结果。

*提示:*
```
这个组中的奇数加起来是一个偶数：4、8、9、15、12、2、1。
回答：加所有奇数（9、15、1）得到25。答案是错误的。

这个组中的奇数加起来是一个偶数：17、10、19、4、8、12、24。
回答：加所有奇数（17、19）得到36。答案是正确的。

这个组中的奇数加起来是一个偶数：16、11、14、4、8、13、24。
回答：加所有奇数（11、13）得到24。答案是正确的。

这个组中的奇数加起来是一个偶数：17、9、10、12、13、4、2。
回答：加所有奇数（17、9、13）得到39。答案是错误的。

这个组中的奇数加起来是一个偶数：15、32、5、13、82、7、1。
回答： 
```

*输出：*
```
加所有奇数（15、5、13、7、1）得到41。答案是错误的。
```

哇！我们发现通过提供推理步骤，我们得到了一个完美的结果。事实上，我们可以通过提供更少的示例来解决这个任务，即仅一个示例似乎就足够了：

*提示:*
```
这个组中的奇数加起来是一个偶数：4、8、9、15、12、2、1。
回答：加所有奇数（9、15、1）得到25。答案是错误的。

这个组中的奇数加起来是一个偶数：15、32、5、13、82、7、1。
回答： 
```

*输出：*
```
加所有奇数（15、5、13、7、1）得到41。答案是错误的。
```请记住，作者声称这是一种随着足够大的语言模型而出现的新兴能力。

## Zero-shot COT Prompting

<Screenshot src={ZEROCOT} alt="Zero-shot COT" />

图片来源：[Kojima等人（2022）](https://arxiv.org/abs/2205.11916)

最近提出的一个新想法是 [zero-shot CoT](https://arxiv.org/abs/2205.11916)（Kojima等人，2022年），它基本上是在原始提示中加入“让我们一步一步地思考” 。 让我们尝试一个简单的问题，看看模型的表现如何：

*提示：*
```
我去了市场并买了10个苹果。 我给了2个苹果给邻居和2个给修理工。 然后我去买了5个苹果并吃了一个。 我还剩下多少个苹果？
```

*输出：*
```
11个苹果
```

答案是不正确的！ 现在来试试特殊提示。

*提示：*
```
我去了市场并买了10个苹果。 我给了2个苹果给邻居和2个给修理工。 然后我去买了5个苹果并吃了一个。 我还剩下多少个苹果？

让我们一步一步地思考。
```

*输出：*
```
首先，您开始有10个苹果。
您给了2个苹果给邻居和2个苹果给修理工，所以您还剩下6个苹果。
然后您买了5个苹果，现在你有11个苹果。
最后，您吃了1个苹果， 因此你还剩下10个苹果。
```

令人印象深刻的是，这个简单的提示对于这个任务非常有效。 这在您没有太多示例用于提示时尤其有用。
